# КТ 2. Анализ зависимостей в проекте с использованием BFS, DFS и топологической сортировки

## Возможности
- Построение графа зависимостей между компонентами.
- Проверка циклов и топологическая сортировка.
- Поиск зависимостей BFS и DFS.
- Вычисление критического пути с учётом весов рёбер.
- Визуализация графа с помощью Graphviz.


## Установка и запуск
pip install -r requirements.txt

python main.py


## Описание
1. Построение графа зависимостей

- Входные данные:

  - Файл с перечислением компонентов и их зависимостей (например, "A зависит от B и C").  

- Структура:

  - Ориентированный граф (рёбра указывают направление зависимости "A → B").  

2. Топологическая сортировка (проверка на циклы)

- Задача:

  - Если граф содержит циклы (например, "A → B → C → A"), проект нельзя собрать.  

  - Реализовать алгоритм топологической сортировки для проверки.  

- Вывод:

  - Либо порядок сборки (топологический порядок), либо сообщение об ошибке "Обнаружен цикл".  

3. Поиск зависимостей (BFS и DFS)

- BFS:

  - Находит зависимости по уровням:  

    - Уровень 1: прямые зависимости.  

    - Уровень 2: зависимости зависимостей и т.д.  

  - Полезно для оценки "глубины" влияния изменения компонента.  

- DFS:

  - Рекурсивно обходит все возможные пути зависимостей.  

  - Полезно для полного перечня всех связанных компонентов.
